Backend Tasks

// Go 1.24 · Fiber · PostgreSQL 15 · Kafka · Progress Tracking

● Backend Architecture & Constraints

  • Stack: Go 1.24, Fiber for REST/WebSocket, PostgreSQL 15, Redis 7, Kafka for events.
  • Monolithic single binary deployment via Docker.
  • Strict Modular Monolith: `modules//` must contain `types.go`, `repository.go`, `service.go`, `handler.go`, `module.go`.
  • No cross-module imports allowed. Use `contracts.Provider` interfaces.
  • All DB migrations in `backend/migrations/` using `.up.sql` and `.down.sql`.

● Progress Tracking & Definition of Done

  • Tracking managed via GitHub Issues or Notion Kanban. Prefix branch names: `feat/` or `fix/`.
  • Unit test coverage must be ≥70% for the service layer.
  • All APIs must have Swagger annotations (`// @Summary`, `// @Tags`, etc.).
  • "Done" means: PR reviewed, tests passing (`go test ./...`), migrations verified, `golangci-lint` passes.
