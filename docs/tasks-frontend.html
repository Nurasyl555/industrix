Frontend Tasks

// React / Next.js · TypeScript · API Client Integration · Progress Tracking

● Frontend Architecture & Constraints

  • Web App uses React/Next.js with TypeScript for SSR on catalog pages.
  • Mobile App uses React Native (iOS + Android) with push notifications and camera integration.
  • Integration strictly through the Go backend monolithic API (REST / WebSocket). No direct DB access.
  • UI must match Figma specifications defined by the design team.
  • Follow the component structure found in `frontend/src/components/`.

● Progress Tracking & Definition of Done

  • Tasks must be tracked via the project management board (e.g., Notion).
  • Each UI component must have a corresponding Storybook entry or visual test.
  • Create Pull Requests mapping to specific `FE-###` task IDs.
  • "Done" means: Figma matched exactly, TypeScript strict mode passes, linter passes, tested against staging API.
