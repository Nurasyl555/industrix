Design Tasks

// Figma · KZ Marketplace · Mobile First · Progress Tracking

● Design Architecture & Constraints

  • Primary tool is Figma. Use variables for color, typography, spacing, and grid systems.
  • Design for both desktop web and mobile application (mobile-first responsive design).
  • Include all interaction states: Default, Hover, Active, Disabled, Loading, Empty.
  • Ensure Cyrillic-compatible fonts (RU+KK support) and appropriate number formatting (e.g. 1 250 000).
  • Follow the existing "Designer Task Plan" (`design-plan.html`).

● Progress Tracking & Definition of Done

  • Tracking managed via Notion Kanban board (Imported from `design-tasks-notion.csv`).
  • Status flow: Not Started → In Progress → In Review → Done → Handed Off.
  • "Done" means: Figma link attached, dev handoff notes written, PM approved, component library updated.
  • For dev handoff, include a Figma comment tagging the developer: `@devname ready for handoff`.
