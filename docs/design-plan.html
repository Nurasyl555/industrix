PHASE 1 Foundation — Design System + Auth Screens
15 tasks · ~18 daysSprint 1▼
DS-001–002
Color tokens, Typography, Spacing + Grid
DESIGN SYSTEMMUST3 daysDesigner A
▼
Color system
Use Figma Variables (not just styles) for full token system: brand colors, semantic (success/error/warning/info), 5 background levels, 4 text levels, 3 border levels. Must support light + dark mode via variable modes.
Typography scale
Display, H1–H3, Body-L, Body, Body-S, Caption, Mono. Define line-height and letter-spacing per style. Include Cyrillic-compatible font (RU+KK text must render correctly).
Handoff for devs
Export token names as CSS custom properties. Variable names must match the codebase convention already established: --bg, --text, --border, --bright, --dim. Spacing scale: 4px base unit (4, 8, 12, 16, 24, 32, 48, 64). Grid: 12 columns, 24px gutter, 16px margin on mobile.

Empty states checklist (DS-070)
Design all: catalog no-results, search no-results, favorites empty, deals inbox empty, notifications empty, reviews empty, documents empty. Each: illustration + 6-word max headline + supporting text + 1 CTA.
i18n check (DS-076)
Russian text is 30% longer on average. Test every button, label, toast, nav item with RU copy — fix overflow. Kazakh (KK) uses Cyrillic — verify same font renders it. Check number formatting: KZ uses space as thousands separator (1 250 000).
Accessibility (DS-075)
Contrast: text min 4.5:1, UI elements min 3:1. Focus states: visible ring on every interactive element. Errors: never conveyed by color alone — always include text. Icon buttons: document aria-label for every one. Form required fields: not just asterisk — aria-required.
Figma library cleanup (DS-074)
All components in "IEM Components" library with proper naming: Component/Variant/State. Auto-layout everywhere. Publish library. Write 1-page usage guide per component group (dos/don'ts). Archive unused exploratory frames.
